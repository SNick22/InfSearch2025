class="tm-article-presenter__origin-link" href="https://amir-ansari.medium.com/java-vs-kotlin-1a119beb43b8" target="_blank">Автор оригинала: <span>Amir Ansari</span></a></div><div class="tm-article-body" data-gallery-root lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/cab/568/158/cab5681587e3de89e4b73e0c62a516de.png" width="780" height="439" data-src="https://habrastorage.org/getpro/habr/upload_files/cab/568/158/cab5681587e3de89e4b73e0c62a516de.png"/><figcaption></figcaption></figure><p>Предлагаю начать с общих черт.</p><ul><li><p>Статическая типизация</p></li><li><p>Бесплатны и с открытым кодом</p></li><li><p>Код преобразуется в байт-код</p></li><li><p>Интероперабельность</p></li><li><p>Объектно-ориентированные языки программирования</p></li></ul><p><strong>1. Статическая типизация</strong></p><p>Java и Kotlin являются языками со статической типизацией. Это означает, что проверка типов выполняется во время компиляции. (Существуют также языки с динамической типизацией, например PHP, Python и JavaScript.)</p><p>Это хорошо видно из примера ниже.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/59e/574/d31/59e574d315e543e21ee3944dfde07517.png" width="947" height="212" data-src="https://habrastorage.org/getpro/habr/upload_files/59e/574/d31/59e574d315e543e21ee3944dfde07517.png"/><figcaption></figcaption></figure><p><strong>2. Бесплатны и с открытым кодом</strong></p><p>Оба языка являются бесплатными и распространяются как открытое программное обеспечение (можно использовать без ограничений и участвовать в развитии).</p><p><strong>3. Байт-код</strong></p><p>Оба языка преобразуют код в байт-код, исполняемый JVM.</p><p><strong>4. Интероперабельность</strong></p><p>Оба языка являются интероперабельными. Это означает, что файлы Java и Kotlin могут сосуществовать в одном проекте или пакете JAR.</p><p><strong>5. Поддержка ООП</strong></p><p>Оба являются объектно-ориентированными языками программирования.</p><p>(Таким образом, они поддерживают основные концепции ООП)</p><p>Полиморфизм, наследование, инкапсуляция, абстракция.</p><p><strong>А теперь рассмотрим основные отличия.</strong></p><p><strong>1. Представление публике и релиз</strong></p><ul><li><p>Язык Java был разработан компанией Sun Microsystems (в настоящее время принадлежит Oracle) в 1995 году.</p></li><li><p>Он поддерживается практически всеми типами устройств и операционных систем, будь то Android, Windows или Linux.</p></li><li><p>Язык Kotlin был представлен компанией JetBrains в 2011 году, выложен в открытый доступ в 2012 году, официально поддержан на Google I/O (ежегодном мероприятии разработчиков Google) в 2017 году.</p></li><li><p>По заявлению Google, 70 % из 1000 лучших приложений для Android сейчас написаны на Kotlin.</p></li><li><p>Некоторые приложения пока находятся в процессе переноса с Java на Kotlin, например приложение Google Home еще не полностью переписано на Kotlin...</p></li><li><p>...но по состоянию на июнь 2020 года около 30 % старой кодовой базы на Java уже было переписано на Kotlin.</p></li><li><p>Другими популярными примерами Kotlin-приложений от Google являются Maps, Play и Drive.</p></li><li><p>Прочие компании также выпустили множество приложений для Android, написанных на языке Kotlin.</p></li><li><p>Сейчас при поддержке разработки под Android компания Google придерживается стратегии «Kotlin прежде всего». Это в чем-то похоже на ситуацию с разработкой приложений под iOS, где произошел переход от Objective-C к Swift.</p></li></ul><p><strong>2. Версия</strong></p><ul><li><p>По состоянию на ноябрь 2020 года актуальной версией Kotlin является 1.4.0.</p></li><li><p>Несмотря на выход Java 15, самой популярной версией все еще остается Java 8 (или 1.8).</p></li></ul><p><strong>3. Скорость</strong></p><ul><li><p>По этому параметру Java превосходит Kotlin на 12–15 % для чистых сборок (то есть в этом случае Kotlin компилируется немного медленнее).</p></li><li><p>Однако в случае частичных сборок с включенной инкрементной компиляцией (то есть с компиляцией только небольших изменений) Kotlin компилируется так же быстро, как и Java, или даже немного быстрее.</p></li></ul><p><strong>4. Количество строк кода</strong></p><ul><li><p>Код, написанный на Kotlin, намного компактнее по сравнению с Java — на 30–40 %.</p><p>Таким образом, в теории размер приложений может уменьшиться на треть.</p></li></ul><ul><li><p>Язык Java — крайне многословный, а Kotlin — лаконичный и современный.</p></li></ul><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/58a/cd9/22e/58acd922e0d554990d29565db2c0b221.png" width="947" height="250" data-src="https://habrastorage.org/getpro/habr/upload_files/58a/cd9/22e/58acd922e0d554990d29565db2c0b221.png"/><figcaption></figcaption></figure><p><strong>5. Доля рынка</strong></p><ul><li><p>По данным опросов, разработчиков на Kotlin примерно в пять раз меньше, чем использующих Java.</p></li><li><p>7,8 % разработчиков предпочитают Kotlin, более чем 40 % — Java. Однако эти опросы также показывают, что Kotlin нравится программистам больше, чем Java, и быстро наращивает аудиторию.</p></li></ul><details class="spoiler"><summary>Ссылки</summary><div class="spoiler__content"><p><a href="https://www.statista.com/statistics/793628/worldwide-developer-survey-most-used-languages/"><em><u>https://www.statista.com/statistics/793628/worldwide-developer-survey-most-used-languages/</u></em></a><em><u><br/></u></em><a href="https://insights.stackoverflow.com/survey/2020#most-popular-technologies"><em><u>https://insights.stackoverflow.com/survey/2020#most-popular-technologies</u></em></a><em><u><br/></u></em><a href="https://insights.stackoverflow.com/survey/2020#most-loved-dreaded-and-wanted"><em><u>https://insights.stackoverflow.com/survey/2020#most-loved-dreaded-and-wanted</u></em></a></p></div></details><p><strong>6. Безопасная работа с обнуляемыми переменными (null safety)</strong></p><ul><li><p>Kotlin защищен от <code>NullPointerException</code>. Именно этот тип ошибки является самой частой причиной сбоев приложений из Google Play.</p></li><li><p>Java позволяет разработчикам присваивать значение <code>null</code> любой переменной.</p></li><li><p>В отличие от Java, в Kotlin по умолчанию все типы являются non-nullable, то есть не могут принимать значение <code>null</code>. Присвоение или возврат <code>null</code> приведет к ошибке компиляции.</p></li><li><p>Чтобы присвоить переменной значение <code>null</code>, в Kotlin необходимо явно пометить эту переменную как <code>nullable</code>.</p></li></ul><pre><code class="kotlin">val number: Int? = null //Nullable type
val name: String = null //Error because not possible to assign a null value</code></pre><p>Nullable-типы используются с оператором безопасного вызова.</p><pre><code class="kotlin">name?.getLength()</code></pre><p>Таким образом, даже если name примет значение <code>null</code>, все выражение будет эквивалентно null без возникновения <code>NullPointerException</code>.</p><p><strong>7. Гибридные приложения</strong></p><ul><li><p>Kotlin можно использовать для написания нативных приложений для Android и iOS.</p></li><li><p>Kotlin Multiplatform Mobile (KMM) работает в Android и iOS.</p></li><li><p>Java до последнего времени не использовалась при разработке приложений под iOS.</p></li></ul><p>Теперь рассмотрим СТИЛЕВЫЕ отличия.</p><ol><li><p><strong>Функция main</strong></p></li></ol><ul><li><p>В Java метод <code>main</code> должен размещаться внутри класса. Он объявляется как статический метод.</p></li><li><p>В Kotlin, чтобы сделать функцию статической, достаточно поместить ее на уровень пакета. Так что это может быть самостоятельная функция без класса.</p></li><li><p>Аргументы метода <code>main</code> в Kotlin можно опустить, если наша программа не должна принимать аргументы командной строки.</p></li></ul><pre><code class="kotlin">fun main(args : Array&lt;String&gt;) {
println(“hello”)
}</code></pre><ul><li><p>В Java, если мы не включаем аргументы в функцию main (даже если мы их не используем), выводится ошибка.</p></li></ul><pre><code>Error: Main method not found in class</code></pre><p><strong>2. Поведение по умолчанию</strong></p><ul><li><p>В отличие от Java, по умолчанию классы в Kotlin являются финальными (final), поэтому, чтобы разрешить наследование от класса, его следует пометить ключевым словом <code>open</code>.</p></li><li><p>Чтобы разрешить переопределение метода, его необходимо явно пометить как open.</p></li></ul><pre><code class="kotlin">class A { … } in Java is equal to open class A { … } in Kotlin.
final class B { … } in Java is equal to class B { …} in Kotlin.</code></pre><ul><li><p>В Kotlin все, что не имеет модификаторов доступа, по умолчанию является <code>public</code>. Мы можем явно прописать <code>public</code> в определении, но это не обязательно.</p></li></ul><pre><code class="kotlin">public class A { … } и class A { … }</code></pre><p>— это одно и то же в Kotlin.</p><p>В Kotlin используется четыре модификатора видимости: <code>private</code>, <code>protected</code>, <code>public</code> и <code>internal</code>.</p><p>Internal подразумевает повсеместную видимость в одном модуле.</p><ul><li><p>В Java используется ключевое слово <code>default</code>.</p></li><li><p>Ключевое слово <code>default</code> является модификатором доступа. Если вы не назначили переменным, методам, конструкторам и классам какой-либо модификатор доступа, то по умолчанию используется модификатор доступа <code>default</code>.</p></li><li><p><code>Default</code> обеспечивает видимость в пределах пакета.</p></li><li><p>Методы с модификатором <code>default</code> позволяют добавлять в интерфейсы новые функциональные возможности.</p></li></ul><pre><code class="java">interface AnInterface {
public default void myMethod() {
System.out.println(“D”);
}
} /Allowed</code></pre><ul><li><p>В Kotlin ключевое слово <code>default</code> не используется.</p></li></ul><p><strong>3. Типы данных и массивы</strong></p><ul><li><p>Изменяемым (mutable) и неизменяемым (immutable) типами в Kotlin являются <code>var</code> и <code>val</code></p></li></ul><pre><code class="kotlin">//Компилятор может определить тип переменной по ее значению.

var website = “hello”
var website: String = “hello” //одно и то же

//Указывать тип обязательно, если вначале идет объявление, а потом инициализация

var website: String
website = “hello“</code></pre><ul><li><p>В числовых литералах разрешается использовать символы подчеркивания.</p></li></ul><pre><code class="kotlin">val creditCardNumber = 1234_5678_9012_3456L</code></pre><p>Нельзя сравнивать типы разной величины.</p><pre><code class="kotlin">val a: Int = 10; val b: Long = 10L

print(a == b) // Ошибка в Kotlin: сравнение невозможно. В Java возвращается true.</code></pre><ul><li><p>Имена примитивных типов данных в Kotlin начинаются с заглавной буквы, например <code>Boolean</code> и <code>Int</code>, в то время как в Java — со строчной, например <code>char</code> и <code>double</code>.</p></li><li><p>Классы-обертки, такие как <code>Integer</code>, доступны в обоих языках.</p></li></ul><pre><code class="kotlin">val num:Integer = Integer(10) //корректный код в Kotlin
val num:Integer = 10 //не будет работать</code></pre><p>Массивы объявляются следующим образом:</p><pre><code class="kotlin">Int[] numbers = new int[] {10,20,30,40,50}
val numbers = intArrayOf(10,20,30,40,50)
val numbers = arrayOf(10,20,30,40,50)
var numbers = IntArray(5){it*10}
var numbers = Array&lt;Int&gt;(5){it*10}</code></pre><p><strong>4. Списки</strong></p><p>Тип List по умолчанию в Kotlin является неизменяемым, поэтому методы <code>add()</code> или <code>remove()</code> работают не так, как в Java.</p><pre><code class="kotlin">val lst = listOf&lt;Int&gt;(10, 20, 30, 40, 50)
lst.add(60) //Ошибка
val lst2 = mutableListOf&lt;Int&gt;(10, 20, 30, 40, 50) //то же самое, что ArrayList&lt;Int&gt;
// val для mutableList? Да, потому что нового присваивания не происходит, только изменение содержимого.
lst2.add(60) //OK
lst2 += 70 //тоже OK</code></pre><p>Оперировать списком в языке Kotlin можно с помощью функций <code>take</code> и <code>drop</code>.</p><pre><code class="kotlin">val nums = listOf(0,1,2,3,4,5,6,7)
nums.take(3) // [0,1,2]
nums.drop(3) // [3,4,5,6,7]</code></pre><p><strong>5. Циклы</strong></p><p>В Kotlin доступно несколько вариантов циклов <code>for</code>. В Java последний имеет фиксированную структуру.</p><pre><code class="kotlin">val lst : List&lt;Int&gt; = listOf&lt;Int&gt;(10, 20, 30, 40, 50)
for(item in lst){
println(item)
}
for(item in 0 until lst.size){
println(lst[item])
}
for(item in 0..4){ //оператор диапазона (range)
println(lst[item])
}</code></pre><p>В заключение предлагаю рассмотреть функциональные различия.</p><ol><li><p><strong>Конструкторы</strong></p></li></ol><ul><li><p>В Kotlin доступны два вида конструкторов.</p></li><li><p>Один из них прописывается после имени класса и называется первичным конструктором, а второй прописывается в теле класса и называется вторичным конструктором.</p></li><li><p>В классе могут быть один первичный конструктор и несколько вторичных.</p></li></ul><p>Первичный конструктор не может содержать код. Код инициализации можно поместить в блок <code>init</code>.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/5ed/46e/91d/5ed46e91db852ece028b1bee3ca244b7.png" width="947" height="247" data-src="https://habrastorage.org/getpro/habr/upload_files/5ed/46e/91d/5ed46e91db852ece028b1bee3ca244b7.png"/><figcaption></figcaption></figure><pre><code class="kotlin">fun main(args: Array&lt;String&gt;) {
val per1 = Person(“amir”) //вызов только первичного конструктора
val per2 = Person(“ansari”,20, ‘A’) //если убрать ‘A’ , вторичный конструктор не вызывается, //так как нет значения по умолчанию для blood_group
}</code></pre><p>Вторичный конструктор должен расширять поведение первичного конструктора.</p><pre><code class="kotlin">class Person (var name: String, var age: Int = 18) {
init{
println(“Student has got a name as $name and age as $age”)
}
var blood_group: Char = ‘O’
constructor(_name: String, age: Int, blood_group: Char) : this(_name, age) {
this.blood_group = blood_group
println(“Student name= $_name and age= $age and blood group=$blood_group”)
}
}</code></pre><p><strong>2. Функции-расширения</strong></p><ul><li><p>Kotlin позволяет разработчикам расширять класс, добавляя новую функциональность при помощи функций-расширений.</p></li><li><p>Это поистине замечательное улучшение, поскольку программисту не требуется расширять класс.</p></li><li><p>По сути, функция-расширение — это функция, которая является членом класса, но определена за его пределами.</p></li><li><p>В Java функций-расширений не было.</p></li></ul><pre><code class="java">fun String.countSpaces(): Int {
return this.count { c -&gt; c == ‘ ‘ }
}</code></pre><p><strong>3. Функции высшего порядка</strong></p><p>В языке Kotlin функция, которая может принимать в качестве параметра функцию или лямбда-выражение или же может возвращать функцию, называется функцией высшего порядка (higher-order function).</p><pre><code class="kotlin">// лямбда-выражение
var lambda = {a: Int , b: Int -&gt; a + b }
//функция высшего порядка
fun highfun( lmbd: (Int, Int) -&gt; Unit) { // принимает лямбда-выражение как параметр, ничего не возвращает
var result = lmbd(2,4) // вызывает лямбда-выражение, передавая ему параметры
println(“Сумма двух чисел равна $result”)
}
fun main() {
highfun(lambda) //лямбда-выражение передается как параметр
}</code></pre><p><strong>4. Data-классы</strong></p><ul><li><p>В больших проектах, как правило, используется несколько классов, предназначенных исключительно для хранения данных.</p></li><li><p>Разработчику на Java приходится писать много стандартного, но часто встречающегося кода (так называемый шаблонный код или <code>boilerplate</code>), data-классы в Kotlin позволяют избежать этих дополнительных усилий.</p></li><li><p>В Java-классе для этой цели должны присутствовать геттеры и сеттеры, функции <code>Hashcode()</code>, <code>toString()</code> и <code>equals()</code>. Эквивалентом в Kotlin будет...</p></li></ul><pre><code class="kotlin">data class Person(var name: String, var surname: String, var id: String)</code></pre><p>— это все!</p><p><strong>5. Статические члены</strong></p><p>Как только мы объявляем переменную <code>static</code>, она загружается в память во время компиляции, то есть доступна только одна ее копия. Одиночки (signgleton) и статические члены имеют схожее поведение.</p><p>Ключевое слово <code>static</code> делает компонент частью класса, не связанной с объектом этого класса.</p><p>В концепции ООП то, что не является объектом, существовать не должно.</p><p>В Java все должно объявляться внутри класса. Но в Kotlin все иначе. Компоненты могут объявляться за пределами класса, и это автоматически делает их статическими. Поэтому нам не требуется ключевое слово <code>static</code>.</p><p>В Java статические члены обрабатываются не так, как члены-объекты. Это означает, что для статических членов нам недоступны такие вещи, как реализация интерфейса, помещение экземпляра в ассоциативный список (map) или передача его в качестве параметра методу, который принимает объект.</p><p>В Kotlin <code>static</code> не является ключевым словом и вместо статических членов используются объекты-компаньоны, позволяющие преодолеть вышеуказанные ограничения.</p><ul><li><p>В этом и заключается преимущество.</p></li><li><p>Даже если члены объектов-компаньонов выглядят как статические члены в других языках, во время выполнения они все равно остаются членами экземпляров реальных объектов и могут, например, реализовывать интерфейсы.</p></li></ul><p><strong>6. Асинхронная обработка</strong></p><p>В Java существует множество решений для асинхронной работы: RxJava, AsyncTask (уже официально не поддерживается), обработчики, обратные вызовы...</p><p>Наряду со всеми этими возможностями в Kotlin также имеются корутины (<code>coroutines</code>, также их называют сопрограммами), которые упрощают работу.</p><p>Корутины (или легковесные потоки) не являются отдельными потоками, но несколько корутин могут совместно использовать один поток.</p><p><strong>7. Проверяемые исключения</strong></p><p>Такие исключения, как IOException и <code>FileNotFoundException</code>, присутствуют в Java, но не поддерживаются в Kotlin.</p><ul><li><p>Причина в том, что они ничего не делают, кроме как содержат комментарий в блоке <code>catch</code>.</p></li></ul><p><strong>8. Ленивая загрузка</strong></p><p>В Kotlin модификаторы lateinit и by Lazy позволяют инициализировать значения до их фактического использования.</p><pre><code class="kotlin">val myUtil by lazy {
MyUtil(parameter1, parameter2)
}
lateinit var myUtil: MyUtil</code></pre><p>Вышеприведенный код инициализирует объект MyUtil. Но это будет сделано только при первом использовании <code>myUtil</code>.</p><p>Оба варианта служат одной цели, но по факту сильно отличаются. Один является неизменяемым (<code>val</code>), другой — изменяемым (<code>var</code>).</p><pre><code>@Injectlateinit var myUtil: MyUtil</code></pre><ul><li><p>Ключевое слово <code>lateinit</code><strong> </strong>было специально введено в Kotlin для переменных внедрения зависимостей (DI). Его следует использовать для изменяемых или устанавливаемых извне значений. Это полезно, если мы не хотим инициализировать значение, но при этом хотим избежать проверки на <code>null</code>.</p></li></ul><p>Чтобы проверить, было ли свойство, определенное как <code>lateinit var</code>, уже инициализировано, вызовите метод <code>.isInitialized()</code> для ссылки на это свойство:</p><pre><code>if (foo::bar.isInitialized) {
println(foo.bar)
}</code></pre><p>Переменная <code>lateinit</code> может быть инициализирована из любого места, откуда доступен объект.</p><p>В свою очередь, Java не поддерживает отложенную инициализацию, поэтому значения инициализируются, даже если они не используются.</p><p>Вот некоторые различия между этими двумя мощными языками.</p><p>Благодарю за внимание!</p><hr/><blockquote><p>Материал подготовлен в рамках курса <a href="https://otus.pw/cpHC/">«Подготовка к сертификации Oracle Java Programmer (OCAJP)». </a></p><p>Всех желающих приглашаем на открытый урок <strong>«Конструкторы и блоки инициализации»</strong>.
