id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-1"><div xmlns="http://www.w3.org/1999/xhtml">Это пост заметка о том, как я познакомился с языком Kotlin, какие использовал для этого ресурсы, инструменты, и как я собрал его в jar и запустил в консоли.<br/>
<br/>
В общем-то я PHP программист, немного заком с Java. На днях решил посмотреть Kotlin. Это менее строго типизированный язык от JetBrains до того, что он, по сути, статически типизированный. В этом его основное принципиальное отличие от Java, плюс Kotlin имеет свой приятный синтаксис. Kotlin может быть скомпилирован в JVM или JavaScript.<br/>
<br/>
Приступим к знакомству. Для начала понадобится IDE Idea. <br/>
<a name="habracut"></a><br/>
<a href="http://www.jetbrains.com/idea/" title="http://www.jetbrains.com/idea/">http://www.jetbrains.com/idea/</a><br/>
<br/>
Инструкции по установке там же на сайте. Версия 13+. <br/>
<br/>
Внутри IDE я создал простой проект Java. Далее надо установить плагин от JetBrains для IDE. <br/>
<a href="http://confluence.jetbrains.com/display/Kotlin/Getting+Started" title="http://confluence.jetbrains.com/display/Kotlin/Getting+Started">http://confluence.jetbrains.com/display/Kotlin/Getting+Started</a><br/>
<br/>
Плагин добавляет автозаполнение и возможность создавать Kotlin проекты. <br/>
Так же добавляются инструменты для компиляции Kotlin.<br/>
<br/>
Особенность Kotlin в том, что он полностью совместим с Java. Т.е. код, написанный на Kotlin можно вызывать в Java и наоборот. Следовательно, все инструменты и библиотеки, которые используются для работы с Java, применимы и для Kotlin.<br/>
<br/>
Если проект в IDE был создан не Kotlin изначально, то после установки плагина можно включить поддержку Kotlin. После включения в проект будут доаблены jar библиотеки Kotlin в директорию /lib.<br/>
<br/>
Я включил Kotlin и написал следующий код для теста в папке /src/testing:<br/>
<br/>
<pre><code class="java">//first.kt
package testing

/**
 * Created by seyfer on 26.03.14.
 */

//fun main(args : Array&lt;String&gt;) {
//    println("Hello, world!")
//}

//fun main(args : Array&lt;String&gt;) {
//    if (args.size == 0) {
//        println("Please provide a name as a command-line argument")
//        return
//    }
//    println("Hello, ${args[0]}!")
//}

/*fun main(args : Array&lt;String&gt;) {
    for (name in args)
        println("Hello, $name!")

    val language = if (args.size == 0) "EN" else args[0]
    println(when (language) {
        "EN" -&gt; "Hello!"
        "FR" -&gt; "Salut!"
        "IT" -&gt; "Ciao!"
        "seyfer" -&gt; "seed!"
        else -&gt; "Sorry, I can't greet you in $language yet"
    })
}
</code></pre><br/>
<br/>
В котлин файле комментарии я оставил нарочно. Это примеры со страницы <br/>
<a href="http://confluence.jetbrains.com/display/Kotlin/Hello%2C+world%21" title="http://confluence.jetbrains.com/display/Kotlin/Hello%2C+world%21">http://confluence.jetbrains.com/display/Kotlin/Hello%2C+world%21</a><br/>
<br/>
Для начала я выполнял в IDE файл котлина отдельно, чтобы убедиться, что плагин работает и код компилируется.<br/>
<br/>
Потом я создал java класс:<br/>
<br/>
<pre><code class="java">//first.java
package testing;

/**
 * Created with IntelliJ IDEA.
 * User: seyfer
 * Date: 24.07.13
 * Time: 21:02
 * To change this template use File | Settings | File Templates.
 */
public class first {

    public static void main(String[] args) {

        System.out.print("Hello " + args[0] + "\n");

        TestingPackage.main(args);
    }

}
</code></pre><br/>
<br/>
Тут видно, что я хочу принять аргумент и передать его напрямую в котлин. В котлине же я использовал пример с языками и добавил еще один свой, который зависит от аргумента — моего ник нейма. В Java можно вызывать методы и получать доступ к переменным из котлина путем вызова — ИмяпакетаPackage.имяметода().<br/>
<br/>
Если правильно настроить в IDE выполнение проекта, то можно задать с каким аргументом вызывать программу. Но это очень не удобно, я хотел бы менять аргумент на лету, чтобы отловить ошибки, для тестироавния. Мне нужно было собрать jar для запуска из консоли.<br/>
В проекте не было maven изначально, и я его туда добавил.<br/>
<br/>
Помогла мне в этом вот эта страница.<br/>
<a href="http://confluence.jetbrains.com/display/Kotlin/Kotlin+Build+Tools" title="http://confluence.jetbrains.com/display/Kotlin/Kotlin+Build+Tools">http://confluence.jetbrains.com/display/Kotlin/Kotlin+Build+Tools</a><br/>
Документация котлина достаточно информативна для изучения языка по ней.<br/>
Так же, по поводу построения при помощи maven и gradle, можно посмотреть следующие примеры.<br/>
<a href="https://github.com/JetBrains/kotlin-examples" title="https://github.com/JetBrains/kotlin-examples">https://github.com/JetBrains/kotlin-examples</a><br/>
<br/>

<br/>
Тут используется настройка для kotlin-maven-plugin, где указывается, в какой директории искать файлы.<br/>
Так же, используется maven-jar-plugin. Тут очень важно правильно прописать &lt; <strong>manifest</strong> &gt;, иначе предстоит долго разбираться, почему jar не запускается с ошибкой "<b>No Main Manifest Attribute</b>".<br/>
После билда в директории /target будет лежать jar проекта. В Idea есть отдельное окно maven projects, откуда удобно управлять построением.<br/>
<br/>
Чтобы запустить jar обычно выполняется команда<br/>
<br/>
<pre><code class="bash">java -jar /path/to/file.jar argument1 argument2
</code></pre><br/>
<br/>
В моем случае это не сработало, т.к. я не воспользовался плагином maven, который бы добавил jar библиотеку котлина в билд.<br/>
Следовательно я получаю ошибку<br/>
<br/>
<pre><code class="bash">java -jar target/idea_test-1.0-SNAPSHOT.jar 1
Hello 1
Exception in thread "main" java.lang.NoClassDefFoundError: kotlin/jvm/internal/Intrinsics
	at testing.TestingPackage-first-39a20e2.main(first.kt:27)
	at testing.TestingPackage.main(first.kt:1)
	at testing.first.main(first.java:16)
Caused by: java.lang.ClassNotFoundException: kotlin.jvm.internal.Intrinsics
	at java.net.URLClassLoader$1.run(URLClassLoader.java:366)
</code></pre><br/>
<br/>
Отсюда понятно, что не находится Kotlin библиотека.<br/>
Правильный вызов с указанием jar зависимости такой<br/>
<br/>
<pre><code class="bash">java -cp target/idea_test-1.0-SNAPSHOT.jar:lib/kotlin-runtime.jar testing.first seyfer
Hello seyfer
seed!
</code></pre><br/>
<br/>
Теперь код работает верно. Как видно, я получил аргумент в java, вывел его, далее я передал его в kotlin, и на его основе отработала конструкция when (по смыслу похожа на switch-case).<br/>
<br/>
Для того, чтобы собрать jar с включением зависимых jar в билд, можно воспользоваться плагином maven — maven-dependency-plugin. Или onejar-maven-plugin к проекту <a href="http://one-jar.sourceforge.net/" title="http://one-jar.sourceforge.net/">one-jar</a>.<br/>
<br/>
Я решил использовать one-jar.<br/>
<div class="spoiler"><b class="spoiler_title">Вот что надо добавить к pom.xml</b><div class="spoiler_text"><pre><code class="xml">&lt;plugin&gt;
    &lt;groupId&gt;org.dstovall&lt;/groupId&gt;
    &lt;artifactId&gt;onejar-maven-plugin&lt;/artifactId&gt;
    &lt;version&gt;1.4.5&lt;/version&gt;
    &lt;executions&gt;
        &lt;execution&gt;
            &lt;configuration&gt;
                &lt;binlibs&gt;
                    &lt;fileSet&gt;
                        &lt;directory&gt;${project.build.directory}/../lib/&lt;/directory&gt;
                        &lt;includes&gt;
                            &lt;include&gt;kotlin-runtime.jar&lt;/include&gt;
                        &lt;/includes&gt;
                    &lt;/fileSet&gt;
                &lt;/binlibs&gt;
                &lt;!-- Optional, default is false --&gt;
                &lt;attachToBuild&gt;true&lt;/attachToBuild&gt;
                &lt;!-- Optional, default is "onejar" --&gt;
                &lt;classifier&gt;onejar&lt;/classifier&gt;
            &lt;/configuration&gt;
            &lt;goals&gt;
                &lt;goal&gt;one-jar&lt;/goal&gt;
            &lt;/goals&gt;
        &lt;/execution&gt;
    &lt;/executions&gt;
&lt;/plugin&gt;

&lt;pluginRepository&gt;
    &lt;id&gt;onejar-maven-plugin.googlecode.com&lt;/id&gt;
    &lt;url&gt;http://onejar-maven-plugin.googlecode.com/svn/mavenrepo&lt;/url&gt;
&lt;/pluginRepository&gt;
</code></pre><br/>
</div></div><br/>
<br/>
Далее пересобрать проект.<br/>
<pre><code class="bash">mvn package
</code></pre><br/>
<br/>
В директории /target появится файл имяпроекта.one-jar.jar. Его можно вызвать обычным методом.<br/>
<br/>
<pre><code class="bash">java -jar target/idea_test-1.0-SNAPSHOT.one-jar.jar seyfer
Hello seyfer
seed!
</code></pre><br/>
<br/>
На этом настройка проекта для меня закончена. Далее можно изучать Kotlin по приведенным выше ссылкам на документацию.<br/>
