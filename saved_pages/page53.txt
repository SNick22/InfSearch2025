article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Kotlin Multiplatform — это набор для<a href="https://www.simplilearn.com/important-steps-to-upskilling-your-software-development-teams-article"> разработки программного обеспечения</a>, который значительно лучше других вариантов разработки кроссплатформенных приложений. В этом руководстве по Kotlin multiplatform вы подробно узнаете все об этом наборе.</p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/64c4ca617bfd99146f84e0f2" data-style="" id="64c4ca617bfd99146f84e0f2" width=""></div><h3>Что такое Kotlin Multiplatform?</h3><p>Kotlin multiplatform — это фича, появившаяся в 2017 году вместе с Kotlin 1.2. Данная возможность позволяет писать один и тот же код непосредственно на нескольких платформах, ориентируясь на кроссплатформенную разработку приложений. В ее основе лежит идея подхода "Write Once Run Anywhere" [пиши один раз, запускай везде], но она выводит данную концепцию на гораздо более высокий уровень, позволяя писать код один раз и потом использовать его на нескольких платформах. </p><p>Мультиплатформенное <a href="https://www.simplilearn.com/how-to-learn-programming-article">программирование</a> на Kotlin позволяет разработчикам решить проблему зависимостей от конкретной платформы. Это сокращает время, затрачиваемое на написание и поддержку одного и того же кода для разных платформ. Kotlin Multiplatform дает возможность включать другие платформы, такие как Linux, Windows и веб (<a href="https://www.simplilearn.com/tutorials/javascript-tutorial/introduction-to-javascript">JavaScript</a>), в то время как его подмножество Kotlin multiplatform mobile позволяет разработчикам писать код бизнес-логики для iOS и Android из одной кодовой базы.</p><h3>Как работает Kotlin Multiplatform?</h3><p>Kotlin для iOS, Android и других популярных операционных систем позволяет хранить многократно используемые фрагменты кода для двух и более версий приложений.    </p><p>Возьмем, к примеру, Android и iOS. Общий код бизнес-логики для Android преобразуется в байткод <a href="https://www.simplilearn.com/best-java-programs-article">Java</a>, который может быть выполнен в JVM. В Android есть виртуальная машина, поэтому он может легко его выполнить. В случае iOS виртуальной машины нет, поэтому код преобразуется в нативный, который может быть выполнен на iOS.</p><p>Теперь в компиляторе Kotlin есть фронтенд, который преобразует код в промежуточное представление (intermediate representation - IR), а бэкенд конвертирует это промежуточное представление в машинный код.</p><p>Так, в случае android он преобразует его в байткод java, а в случае iOS - в нативный машинный код iOS.</p><h3>Общий код Kotlin</h3><p>Чтобы обеспечить запуск общего кода, в Kotlin предусмотрены компиляторы под конкретные платформы. Для преобразования кода в соответствии с различными платформами предоставляются различные компиляторы. К ним относятся Kotlin/ JVM, Kotlin/ JavaScript и Kotlin/ Native. Эти версии содержат специфичные для конкретной платформы инструменты, библиотеки и расширения языка Kotlin.</p><figure class=""><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/643/ee8/ec6/643ee8ec6401fe3bdcb87fc9a3d822aa.png" width="355" height="365" data-src="https://habrastorage.org/getpro/habr/upload_files/643/ee8/ec6/643ee8ec6401fe3bdcb87fc9a3d822aa.png"/></figure><p>Каждая платформо-зависимая версия включает различные платформы; например, JVM содержит Java 8, Java 6 и Android, JavaScript - Browser и Node, а Native - iOS, Mac, Windows, Linux и т.д.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d4a/b10/b58/d4ab10b586edaf2c5f881b7218fe5e41.png" width="728" height="313" data-src="https://habrastorage.org/getpro/habr/upload_files/d4a/b10/b58/d4ab10b586edaf2c5f881b7218fe5e41.png"/></figure><p>Мультиплатформенные библиотеки помогают при разработке общих частей проекта в одной области, предоставляя API для общего кода Kotlin и впоследствии распространяя его на все целевые платформы.</p><p>Версии платформо-ориентированных библиотек предоставляют библиотеки, которые могут быть использованы для расширения специфических возможностей целевых платформ.</p><h2>Чем мультиплатформенность отличается от кроссплатформенности?</h2><p>Мультиплатформенная система — в которой программа для разных платформ использует как коды, характерные для каждой из них, так и общий код, используемый обеими платформами.</p><p>Кроссплатформенное приложение — это система, позволяющая выполнять деплой приложения с использованием абсолютно одинакового кода приложения.</p><p>Основной упор в Kotlin multiplatform делается на совместное использование общего кода, и мультиплатформенность рассматривается как развитие нативной разработки.</p><h3>Преимущества Kotlin Multiplatform</h3><ul><li><p>Поддержка и сообщество: Многие компании и организации выбирают язык Kotlin. Кроме того, после того, как компания Google объявила Kotlin предпочтительным для android, его выбирают многие компании. На этом языке начали работать множество пользователей, что означает огромную поддержку Kotlin со стороны сообщества.</p></li><li><p>Быстрый и чистый код: В Kotlin multiplatform нет необходимости заново создавать и переписывать код для разных платформ. Можно просто использовать общий код для разных платформ, что экономит огромное количество времени. Такие функциональные возможности, как интероперабельность, также делают работу очень простой и быстрой, и не возникает отставания от нативных платформ; благодаря этому разработчикам не нужно ждать, пока появятся новые фичи для поддержки.</p></li><li><p>Быстрая разработка: Совместное использование кода экономит массу времени и помогает сделать процесс разработки намного быстрее и проще. Его особенности, включая "нулевую безопасность", поддержку IDE и взаимодействие с Java, делают его менее сложным и быстрым.</p></li><li><p>Один язык для кроссплатформенных и нативных приложений: Большинство средств разработки кроссплатформенных приложений не позволяют использовать возможности, присущие устройствам на базе iOS и Android. В Kotlin нет необходимости внедрять в кодовую базу новый язык программирования. Можно легко заставить ваше android-приложение работать на iOS-устройствах без значительных изменений в коде.</p></li></ul><h3>Компании, выбирающие Kotlin Multiplatform</h3><ul><li><p>Netflix: Популярная OTT-платформа Netflix перестроила текущий пользовательский интерфейс плеера в Android-приложении, используя Kotlin.</p></li><li><p>Autodesk: Autodesk — транснациональная корпорация по разработке программного обеспечения. Kotlin хорошо работает с iOS и android без нативного интерфейса Java, что является основной причиной для компании сделать выбор в пользу Kotlin multiplatform.</p></li><li><p>Vmware: Vmware — американская компания, занимающаяся <a href="https://www.simplilearn.com/tutorials/cloud-computing-tutorial/what-is-cloud-computing">облачными вычислениями</a> и технологиями виртуализации. Команде разработчиков Vmware требовалась кроссплатформенная стратегия для мобильных приложений, поэтому был выбран Kotlin multiplatform, и теперь он является "основным фреймворком по умолчанию" для VMware.</p></li><li><p>Trello: Trello — это приложение для управления проектами. Android-приложение Trello создано с использованием Kotlin. Trello использует Kotlin из-за его совместимости с Java. </p></li><li><p>Philips: Philips — очень популярная транснациональная корпорация, и вы наверняка слышали об этой компании. В Philips Kotlin Multiplatform оказался более быстрым при имплементации новых возможностей, а также позволил добиться более тесного взаимодействия в команде между разработчиками Android и iOS.</p></li><li><p>Ice Rock: Ice Rock, аутсорсинговая компания, предоставляющая мобильные и backend-решения для различных отраслей и сегментов рынка, использует Kotlin multiplatform mobile при создании приложений для своих клиентов.</p></li></ul><h3>Заключение</h3><p>В этом руководстве по Kotlin multiplatform вы поняли, как он работает, и изучили, что это такое. Вы также узнали об общем коде Kotlin и выяснили преимущества Kotlin multiplatform. В Android есть виртуальная машина, поэтому он может легко его выполнить. В случае iOS виртуальной машины нет, поэтому код преобразуется в нативный, который может быть выполнен на iOS.\u003C\u002Fp\u003E\u003Cp\u003EТеперь в компиляторе Kotlin есть фронтенд, который преобразует код в промежуточное представление (intermediate representation - IR), а бэкенд конвертирует это промежуточное представление в машинный код.\u003C\u002Fp\u003E\u003Cp\u003EТак, в случае android он преобразует его в байткод java, а в случае iOS - в нативный машинный код iOS.\u003C\u002Fp\u003E\u003Ch3\u003EОбщий код Kotlin\u003C\u002Fh3\u003E\u003Cp\u003EЧтобы обеспечить запуск общего кода, в Kotlin предусмотрены компиляторы под конкретные платформы.
