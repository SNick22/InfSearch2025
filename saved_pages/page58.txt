<h4>Введение</h4> <br/>
В преддверии выхода языка Kotlin из beta, хочется поделиться своим впечатлением от его использования.<br/>
<br/>
<a href="https://kotlinlang.org/">Kotlin</a> — это новый прекрасный язык от JetBrains (разработчиков IntelliJ Idea) для JVM, Android и браузера, который на первый взгляд выглядит как улучшенная java (или как упрощенная scala). Но это только на первый взгляд, язык не только впитал в себя многие интересные решения от других языков, но и представляет свои оригинальные: <br/>
<br/>
 — optional от swift, <a href="https://kotlinlang.org/docs/reference/null-safety.html">nullsafe в kotlin</a> <br/>
 — case классы из scala, <a href="https://kotlinlang.org/docs/reference/data-classes.html">data class в kotlin</a><br/>
 — замена implicit методам из scala, <a href="https://kotlinlang.org/docs/reference/extensions.html">extension функций</a><br/>
 — <a href="https://kotlinlang.org/docs/reference/delegation.html">делегаты</a><br/>
 — <a href="https://kotlinlang.org/docs/reference/null-safety.html">null safely</a><br/>
 — <a href="https://kotlinlang.org/docs/reference/typecasts.html">smart cast</a><br/>
 — и многое другое, подробнее можно посмотреть на официальном сайте <a href="https://kotlinlang.org/">kotlinlang</a>.<br/>
<br/>
Для тех кто знаком с java или scala, будет интересно сравнение <a href="https://kotlinlang.org/docs/reference/comparison-to-java.html">kotlin &amp; java</a>, <a href="https://kotlinlang.org/docs/reference/comparison-to-scala.html">kotlin &amp; scala</a>.<br/>
<br/>
Авторы языка стремятся добиться двух задач:<br/>
 — сделать скорость компиляции сравнимой с java<br/>
 — язык должен быть достаточно выразительным, и при этом быть простым насколько возможно<br/>
Поэтому, стоит оговориться, что если вы на текущей момент счастливы со scala, с ее «сложностью» и временем компиляции, тогда вам скорее всего не нужен будет kotlin, для всех остальных читать дальше:<br/>
<a name="habracut"></a> <br/>
Для тех кто <s>в танке</s> впервые слышит о языке, ниже несколько примеров с официального сайта:<br/>
<br/>

   println("Hello World!")

      return
   }

</div></div><br/>

Из личного опыта применения kotlin особо хочется отметить несколько преимуществ языка:<br/>
<br/>
 — первое это конечно простоту взаимодействия с java. Все типы и коллекции из java преобразовываются в аналогичные из kotlin, и наоборот. Это особенно радует после всей той «анархии», которая творится в scala (да есть scala.collection.JavaConversions._ и scala.collection.JavaConverters._, но все же это не сравниться с полностью прозрачной конвертацией типов);<br/>
 — также не может не радовать отличная поддержка от студии Intellij Idea, хоть язык и находится в Beta 4, уже на текущий момент плагин для студии позволяет комфортно работать;<br/>
 — а для любителей implicit методов из scala, kotlin преподносит очень удобное решение в виде <a href="https://kotlinlang.org/docs/reference/extensions.html">extension функций</a>;<br/>
 — помимо всего прочего разработчики языка ставят своей целью добиться <b>времени компиляции сравнимой с java</b> (привет scala), за что им только хочется пожать руки! Это особенно сильно радует после долгой работы в scala, когда редактирование одной строчки в достаточно небольшом файле компилируется с той же скоростью что и небольшой проект на java;<br/>
 — <a href="https://kotlinlang.org/docs/reference/inline-functions.html">inline функции</a> — отличное нововведение. С их помощью можно, например, расширить текущие возможности языка, или в некоторых ситуациях добиться повышения производительности;<br/>
 — удобные <a href="http://beust.com/weblog/2015/10/30/exploring-the-kotlin-standard-library/">функции</a> стандартной библиотеки.<br/>
 — удобные лямбды, в отличие от той же java 8. Очень похожи на реализацию из scala.<br/>
<br/>
Тем не менее у языка есть и свои недостатки:<br/>
<br/>
 — не хватает <a href="http://docs.scala-lang.org/tutorials/tour/pattern-matching.html">pattern matching</a> из scala, но в некоторых ситуациях спасает <a href="https://kotlinlang.org/docs/reference/typecasts.html">smart cast</a> и <a href="https://kotlinlang.org/docs/reference/multi-declarations.html#destructuring-declarations">Destructuring Declarations</a>, в других же приходится выкручиваться другими средствами. Отсутствие pattern matching в целом понятно, разработчики стараются добиться максимального приближения к времени компиляции java, но его наличие позволило бы существенно упростить написание некоторых приложений, так что довольствуемся тем что есть;<br/>
 — <a href="https://kotlinlang.org/docs/reference/idioms.html">try with resource</a> пока реализован не очень удачно. Но тут авторы языка обещают в ближайшее время исправить ситуацию. А пока можно либо применять имеющееся решение, либо воспользоваться расширением языка:<br/>
<br/>
<div class="spoiler"><b class="spoiler_title">try-with-resources</b><div class="spoiler_text"><pre><code class="java">internal class ResourceHolder : AutoCloseable {
    val resources = ArrayList&lt;AutoCloseable&gt;()

    fun &lt;T : AutoCloseable&gt; T.autoClose(): T {
        resources.add(this)
        return this
    }

    override fun close() {
        resources.reverse()
        resources.forEach {
            try {
                it.close()
            } catch (e: Throwable) {
                e.printStackTrace()
            }
        }
    }
}

inline internal fun &lt;R&gt; using(block: ResourceHolder.() -&gt; R): R {
    val holder = ResourceHolder()
    try {
        return holder.block()
    } finally {
        holder.close()
    }
}
</code></pre><br/>
</div></div> <br/>
<div class="spoiler"><b class="spoiler_title">Пример использования</b><div class="spoiler_text"><pre><code class="java">fun copy(from: Path, to: Path) {
    using {
        val input = Files.newInputStream(from).autoClose()
        val output = Files.newOutputStream(to).autoClose()
        input.copyTo(output)
    }
}
</code></pre><br/>
</div></div><br/>
 — пока нет async и yield, но по словам авторов, после релиза 1.0 можно ждать их появление в самом ближайшем будущем.<br/>
<br/>
<b>Update </b> В комментариях наибольший интерес вызывает сравнение со scala, и в частности преимущество kotlin:<br/>
<div class="spoiler"><b class="spoiler_title">Преимущества kotlin по сравнению со scala</b><div class="spoiler_text">— взаимодействие с java и обратно. На kotlin вообще можно забыть про то что ты вызываешь что-то из java, все предельно прозрачно и удобно. Тоже самое наоборот. Из java также удобно работать с kotlin. Тут даже не только сходство типов помогает, сколько сама ориентированность kotlin на прозрачную работу с java. Проще всего это продемонстрировать на примерах:<br/>
<div class="spoiler"><b class="spoiler_title">Примеры</b><div class="spoiler_text">1. Использование лямбд вместо анонимных классов (в scala правда тоже скоро это может появиться, но пока такой возможности нет). <br/>
Код в java:<br/>
 <pre><code class="java">ExecutorService executor = Executors.newFixedThreadPool(1);
    executor.execute(System.out::println);
</code></pre> <br/>
<br/>
В scala же уже не получится такое провернуть. А вот с kotlin проблем нет:<br/>
<pre><code class="java">val executor = Executors.newFixedThreadPool(1)
    executor.execute { println() }
</code></pre> <br/>
<br/>
2. Есть тот же try-with-resources (да, можно сделать extension к языку, но из коробки этого нет, а значит придется тянуть решение из одного проекта в другой)<br/>
Это по крайней мере что первое приходит в голову<br/>
</div></div><br/>
 — отсутствие implicit. Сами по себе implicit конечно мощное решение, но без студии разобраться в чужом коде что к чем, почти нереально. Особенно если авторы «сильно увлеклись». А для добавления методов kotlin позволяет писать extension, которые на практике выглядят гораздо удобнее аналогичных решений в scala. Пример:<br/>
Scala:<br/>
<pre><code class="scala">implicit class RichInt(val x: Int) extends AnyVal {
  def square: Int = x * x
}
object App  {
   def print(): Unit = {
      val two = 2
      println(s"The square of 2 is ${two.square}")
  }
}
</code></pre><br/>
Kotlin:<br/>
<pre><code class="java">fun Int.richInt(): Int = this * this

object App {
    fun print(): Unit {
        val two = 2
        println("The square of 2 is ${two.richInt()}")
    }
}
</code></pre><br/>
<br/>
 — nullable типы вместо Option. <br/>
<br/>
 — удобные <a href="http://beust.com/weblog/2015/10/30/exploring-the-kotlin-standard-library/">функции</a> стандартной библиотеки <br/>
<br/>
 — ну и конечно же скорость компиляции. На практике действительно все очень быстро компилируется, особенно по сравнению со scala.<br/>
<br/>
 — результирующий jar меньше аналогичного в scala. <br/>
<br/>
 — и что немаловажно, JetBrains обещают <b>полную обратную совместимость</b> (привет еще раз scala c либами под 2.10, 2.11, 2.12 и т.д.)<br/>
</div></div><br/>
<br/>
Перейдем к примеру, в котором будет продемонстрировано небольшое RESTful приложение на <a href="http://projects.spring.io/spring-boot/">spring boot</a>, со сборкой через <a href="http://gradle.org/">gradle</a>. <br/>
<br/>
<h4>Настройка студии</h4><br/>
Для работы необходимо поставить <a href="https://www.jetbrains.com/idea/#chooseYourEdition">IntelliJ Idea Community</a> (но можно использовать и Eclipse, под нее также есть плагин), в которой после установки обновить плагин kotlin. <b>Обновить его необходимо вручную, через settings -&gt; plugin</b>, даже если вы перед этим выбрали обновление плагина через всплывающее окно (по крайней мере на данный момент, пока язык в beta).<br/>
<br/>
Также лучше поставить локальный gradle, и прописать его в настройках в студии (settings -&gt; build, execution, deployment -&gt; gradle -&gt; user local gradle distribution. После чего указать путь к gradle в gradle home).<br/>
<br/>
<h4>Настройка проекта</h4><br/>
Создаем проект gradle kotlin (new project -&gt; gradle -&gt; kotlin) и изменяем содержимое build.gradle на следующее:<br/>

Создаем файл Application.kt в папке src/main/kotlin/test.kotlin.spring.project. В нем будут основные настройки для запуска spring boot:<br/>
<br/>
<div class="spoiler"><b class="spoiler_title">Application.kt</b><div class="spoiler_text"><pre><code class="java">package test.kotlin.spring.project

import com.fasterxml.jackson.annotation.JsonInclude
import com.fasterxml.jackson.databind.DeserializationFeature
import com.fasterxml.jackson.databind.ObjectMapper
import com.fasterxml.jackson.module.kotlin.registerKotlinModule
import org.springframework.boot.SpringApplication
import org.springframework.boot.autoconfigure.EnableAutoConfiguration
import org.springframework.boot.autoconfigure.SpringBootApplication
import org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
import org.springframework.boot.builder.SpringApplicationBuilder
import org.springframework.boot.context.web.SpringBootServletInitializer
import org.springframework.context.annotation.Bean
import org.springframework.http.converter.json.MappingJackson2HttpMessageConverter

@SpringBootApplication
@EnableAutoConfiguration(exclude = arrayOf(DataSourceAutoConfiguration::class))
open class Application : SpringBootServletInitializer() {

    @Bean
    open fun mapperForKotlinTypes(): MappingJackson2HttpMessageConverter {
        return MappingJackson2HttpMessageConverter().apply { objectMapper = jacksonMapper }
    }

    override fun configure(application: SpringApplicationBuilder): SpringApplicationBuilder =
            application.sources(Application::class.java)

    companion object {

        val jacksonMapper = ObjectMapper().registerKotlinModule()
                .setSerializationInclusion(JsonInclude.Include.NON_ABSENT)
                .enable(DeserializationFeature.FAIL_ON_NULL_FOR_PRIMITIVES)

        @Throws(Exception::class)
        @JvmStatic fun main(args: Array&lt;String&gt;) {
            println("starting application...")
            SpringApplication.run(Application::class.java, *args)
        }
    }
}
</code></pre><br/>

Также необходимо будет создать файл с настройками методов rest сервиса. Будет несколько методов:<br/>
<br/>
 — метод будет выдавать AckResponse на введенные с запроса данные об имени и фамилии. <br/>
 — метод, на вход поступает массив строк, из которого выбирается наименьшая строка по длине, которая потом разбивается по '_', сортируется и собирается в строку уже с символом ',' (демонстрирует возможности языка)<br/>
<br/>
Создаем файл ServiceController.kt в папке src/main/kotlin/test.kotlin.spring.project.<br/>
<br/>
<div class="spoiler"><b class="spoiler_title">ServiceController.kt</b><div class="spoiler_text"><pre><code class="java">package test.kotlin.spring.project

import org.springframework.http.MediaType
import org.springframework.web.bind.annotation.RequestMapping
import org.springframework.web.bind.annotation.RequestMethod
import org.springframework.web.bind.annotation.RequestParam
import org.springframework.web.bind.annotation.RestController

data class AckResponse(val status: Boolean, val result: String, val message: String? = null)

@RestController
class ServiceController {
    @RequestMapping(
            path = arrayOf("/request"),
            method = arrayOf(RequestMethod.GET),
            produces = arrayOf(MediaType.APPLICATION_JSON_UTF8_VALUE))
    fun nameRequest(
            @RequestParam(value = "name") name: String,
            @RequestParam(value = "surname", required = false) surname: String?): AckResponse {
        return if (surname == null)
            AckResponse(status = true, result = "Hi $name", message = "surname is empty")
        else
            AckResponse(status = true, result = "Hi $surname,$name")
    }

    @RequestMapping(
            path = arrayOf("/sort_request"),
            method = arrayOf(RequestMethod.GET),
            produces = arrayOf(MediaType.APPLICATION_JSON_UTF8_VALUE))
    fun findMinimum(
            @RequestParam(value = "values") values: Array&lt;String&gt;): AckResponse {
        println("values:")
        values.forEach { println(it) }

        val minValue = values.apply { sortBy { it.length } }
            .firstOrNull()
            ?.split("_")
            ?.sorted()
            ?.joinToString(",") ?: ""

        return AckResponse(status = true, result = minValue)
    }
}

</code></pre><br/>
</div></div><br/>

После успешного запуска, пробуем открыть страницу <a href="http://localhost:8080/request?name=Kris">Запрос с именем</a>. Ответ должен выглядеть следующем образом:<br/>
<pre><code class="javascript">{
  status: true,
  result: "Hi Kris",
  message: "surname is empty"
}
</code></pre><br/>
И <a href="http://localhost:8080/request?name=Kris&amp;surname=Eagle">Запрос с именем и фамилией</a>, тогда ответ будет немного другой:<br/>
<br/>
<pre><code class="javascript">{
  status: true,
  result: "Hi Eagle, Kris"
}
</code></pre><br/>
Вызов для проверки сортировки данных: <a href="http://localhost:8080/sort_request?values=virst_value_1_3,second_value_2_5_6">Сортировка</a>. В результате должно быть:<br/>
<br/>
<pre><code class="javascript">{
  status: true,
  result: "1,3,value,virst"
}
</code></pre><br/>
Тот же вызов, но с пустым массивом: <a href="http://localhost:8080/sort_request?values">Вызов</a><br/>
<pre><code class="javascript">{
  status: true,
  result: ""
}
</code></pre><br/>
При необходимости можно собрать весь проект в один runnable jar, командой: gradle build. В результате проект будет собран в один архив, содержащий все зависимости без распаковки. При таком подходе существенно повышается время сборки проекта, по сравнению с тем же assemble, когда проект собирается в один архив с распаковкой всех зависимостей. <br/>

<h4>Заключение</h4><br/>
В заключении хочется отметить что kotlin оказался весьма удобным языком для работы над любым проектом, где используется java, в качестве ее замены. Экосистема языка пока не такая обширная как та же scala, но уже сейчас можно использовать его в том же big data, где есть java api. К тому же из kotlin очень просто взаимодействовать с java, так что все что есть в java, можно использовать и в kotlin. К тому же из студии есть возможность легкой конвертации java файлов в аналогичные на kotlin (правда нужно будет немного руками подправить файл после конвертации). JetBrains проделали замечательную работу на пути создания идеального языка на смену java и scala. И надеюсь в будущем тенденция в сторону использования kotlin будет только расти.<br/>
