class="tm-article-body" data-gallery-root lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Kotlin Multiplatform — технология, позволяющая объединять бизнес-логику для приложений разных платформ. В ней доступен полный контроль над тем, какие нативные инструменты использовать, а какие вынести в общий модуль (shared). Это позволяет применять данную технологию в уже существующих проектах, что существенно отличает Kotlin Multiplatform от других кроссплатформенных фреймворков таких, как Cordova или Flutter.</p><p>Использование приложениями общего модуля Kotlin Multiplatform позволяет:</p><ul><li><p>Дополнить привычный для платформы функционал.</p></li><li><p>Стандартизировать подходы.</p></li><li><p>Упорядочить конфигурационные файлы.</p></li><li><p>Упростить написание приложений.</p></li></ul><p>Приложение на базе ОС Аврора может работать через компонент QML <a href="https://developer.auroraos.ru/doc/software_development/reference/webview" rel="noopener noreferrer nofollow">WebView</a>, поставляемый ОС Аврора, на базе Kotlin Multiplatform JS. На данный момент нет нативной поддержки Kotlin Multiplatform, информацию о доступных фреймворках можно найти по <a href="https://youtu.be/s0AraORVyNw" rel="noopener noreferrer nofollow">ссылке</a>. Доступен также способ разработки кроссплатформенного приложения с использованием Flutter, информацию по статусу развития можно найти <a href="https://www.youtube.com/live/G-EHh794OB8?t=1782" rel="noopener noreferrer nofollow">здесь</a>.</p><p>При наличии у разработчика готового приложения, написанного на Kotlin Multiplatform (KMP), оно может быть портировано на ОС Аврора. Либо можно написать новое приложение с использованием Kotlin Multiplatform JS под ОС Аврора.</p><p>В данной статье показаны основные этапы портирования существующего приложения Kotlin Multiplatform на ОС Аврора. Портирование выполняется через <a href="https://kotlinlang.org/docs/multiplatform-full-stack-app.html#targets" rel="noopener noreferrer nofollow">цель сборки</a> JS (Target JS). Для демонстрации используется открытое приложение для чтения RSS-новостей <a href="https://github.com/Kotlin/kmm-production-sample" rel="noopener noreferrer nofollow">KMM RSS Reader</a>. Портированное приложение можно найти по <a href="https://gitlab.com/omprussia/kmp/kmm-rss-reader" rel="noopener noreferrer nofollow">ссылке</a>.</p><p>Схема взаимодействия приложения с общим модулем может выглядеть следующим образом:</p><figure class=""><img src="https://habrastorage.org/r/w1560/webt/7i/iz/3o/7iiz3ome8il6y4pua5dqeo9-a74.png" alt="Схема взаимодействия приложения с общим модулем" data-src="https://habrastorage.org/webt/7i/iz/3o/7iiz3ome8il6y4pua5dqeo9-a74.png"/><div><figcaption>Схема взаимодействия приложения с общим модулем</figcaption></div></figure><p>Компонент Wrapper является связующим модулем между общим модулем Kotlin Multiplatform и приложением Qt/QML. Wrapper нужен для того, чтобы обеспечить работу с асинхронными функциями из QML, не модифицируя общий модуль KMP.</p><h3>Структура</h3><p>Приложение будет состоять из трех основных элементов:</p><ul><li><p>Wrapper — библиотека JavaScript, которая связывает общий модуль Kotlin Multiplatform и QML.</p></li><li><p>QML-компонент (KMMAgent) — QML-обёртка над WebView, которая связывает нативный код QML и Wrapper.</p></li><li><p>Kotlin Multiplatform — общий модуль, который реализует бизнес-логику.</p></li></ul><blockquote><p>Примечание.<br/> В связи <a href="https://blog.jetbrains.com/kotlin/2023/07/update-on-the-name-of-kotlin-multiplatform/" rel="noopener noreferrer nofollow">признанием</a> устаревшим названием продукта "Kotlin Multiplatform Mobile" в статье используется новая аббревиатура Kotlin Multiplatform - KMP.<br/> Название портируемого приложения (KMM RSS Reader) и компонента (KMMAgent) остается прежним.</p></blockquote><p>Схема взаимодействия элементов выглядит следующим образом:</p><figure class=""><img src="https://habrastorage.org/r/w1560/webt/vi/9w/tz/vi9wtzk_d-smzopbwxfhvxsfllg.png" alt="Схема взаимодействия элементов" data-src="https://habrastorage.org/webt/vi/9w/tz/vi9wtzk_d-smzopbwxfhvxsfllg.png"/><div><figcaption>Схема взаимодействия элементов</figcaption></div></figure><p>Kotlin Multiplatform собирается в библиотеку Kotlin Multiplatform JS, которую можно подключить к любым проектам JS. Собранная библиотека Kotlin Multiplatform JS подключается к JS-проекту Wrapper, который обеспечивает работу асинхронных функций через <code>Promise</code> с помощью событий <code>events</code>. Собранный webpack Wrapper подключается в файле <code>index.html</code> (файл <code>index.html</code> подключается локально как точка входа для QML-компонента). Далее QML-компонент вызывает функции Wrapper и слушает события <code>events</code> с помощью <a href="https://developer.auroraos.ru/doc/software_development/reference/webview" rel="noopener noreferrer nofollow">WebView</a>, которые приходят из библиотеки Wrapper. WebView выполняет роль обслуживания бизнес-логики и скрыт от пользователя.</p><p>Структура директорий может быть выстроена на усмотрение разработчика с учетом платформы. Важно понимать, что три компонента (Kotlin Multiplatform, Wrapper, приложение Аврора) собираются отдельно, но в конечном итоге JavaScript-библиотека Wrapper должна быть доступна приложению Аврора при сборке, для этого она должна находится в разделе <code>qml</code>.</p><p>Шаги по портированию KMP-приложения на платформу ОС Аврора могут быть следующими:</p><ol><li><p>Подготовить исходный приложения созданного с помощью Kotlin Multiplatform. Например, <a href="https://github.com/Kotlin/kmm-production-sample" rel="noopener noreferrer nofollow">KMM RSS Reader</a>.</p></li><li><p>Собрать общий модуль KMP из исходного кода, как NPM-пакет, с помощью Gradle-плагина <code>npm-publish</code>.</p></li><li><p>Создать проект приложения Qt/QML для ОС Аврора, используя <a href="https://developer.auroraos.ru/doc/software_development/sdk" rel="noopener noreferrer nofollow">Aurora SDK</a>.</p></li><li><p>Добавить в проект QML-компонент на основе WebView для обеспечения работы с асинхронными функциями.</p></li><li><p>Создать JS-библиотеку Wrapper, которая свяжет KMP и QML-компонент.</p></li><li><p>Приступить к разработке пользовательского интерфейса приложения под ОС Аврора с использованием бизнес-логики Kotlin Multiplatform.</p></li></ol><h3>Сборка общего модуля Kotlin Multiplatform</h3><p>Для добавления JavaScript в Kotlin Multiplatform нужно собрать модуль как JavaScript-библиотеку. В приведенном примере используется версия <code>1.8.0</code> плагина <code>multiplatform</code>. Подробная информация по интеграции целей сборки JS доступна в документации <a href="https://kotlinlang.org/docs/js-ir-compiler.html" rel="noopener noreferrer nofollow">"Kotlin/JS IR compiler"</a>. Сначала требуется добавить секцию <code>js</code> в <code>kotlin {}</code> в конфигурационный файл Gradle:</p><p>Файл <code>&lt;проект&gt;/shared/build.gradle.kts</code></p><pre><code class="kotlin">js(IR) {
    moduleName = "shared"
    version = "0.0.1"
    nodejs()
    binaries.library()
}
</code></pre><p>Для сборки модуля Kotlin Multiplatform как JS npm-пакета можно использовать Gradle-плагин <a href="https://github.com/mpetuska/npm-publish" rel="noopener noreferrer nofollow"><code>npm-publish</code></a>. Его можно добавить следующим образом:</p><p>Файл <code>&lt;проект&gt;/shared/build.gradle.kts</code></p><pre><code class="kotlin">plugins {
    id("dev.petuska.npm.publish") version "3.3.1"
}

npmPublish {
    packages {
        named("js") {
            packageJson {
                version.set("0.0.1")
            }
        }
    }
}
</code></pre><p>Плагин <code>npm-publish</code> добавит метод <code>packJsPackage</code>. Его можно вызвать из командной строки для сборки npm-пакета:</p><pre><code class="bash">./gradlew packJsPackage
</code></pre><h3>QML-компонент (KMMAgent)</h3><p>QML-компонент будет содержать <a href="https://developer.auroraos.ru/doc/software_development/reference/webview" rel="noopener noreferrer nofollow">WebView</a>. WebView по умолчанию может работать только с синхронными функциями. Поэтому нужно добавить к WebView поддержку <code>Promise</code>.</p><blockquote><p>Объект Promise представляет возможное завершение (или сбой) асинхронной операции и ее результирующее значение.</p><p><a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise" rel="noopener noreferrer nofollow">Promise</a></p></blockquote><p>Портируемое приложение будет обращаться в сеть для запроса XML-данных RSS. Так как запросы будут выполняться через WebView, следует отключить CORS.</p><blockquote><p>CORS (Cross-origin resource sharing) — это механизм, который позволяет запрашивать ограниченные ресурсы на веб-странице из другого домена за пределами домена, из которого обслуживался первый ресурс.</p><p><a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS" rel="noopener noreferrer nofollow">Cross-Origin Resource Sharing (CORS)</a></p></blockquote><p>Далее нужно создать скрипт framescript.js в приложении ОС Аврора. Он позволит слушать события <code>events</code> приходящие со стороны библиотеки Wrapper.</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/shared-js/framescript.js</code></p><pre><code>addEventListener("DOMContentLoaded", function (aEvent) {
    aEvent.originalTarget.addEventListener("framescript:log",
        function (aEvent) {
            sendAsyncMessage("webview:action", aEvent.detail)
    });
});
</code></pre><p>Скрипт framescript.js нужно подключить к WebView, на основе которого создается QML-компонент <code>KMMAgent</code>.</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/pages/KMMAgent.qml</code></p><pre><code>WebView {
    id: webview

    height: 0
    width: 0
    url: Qt.resolvedUrl("../shared-js/index.html")
    visible: false

    onViewInitialized: {
        // Подключение слушателя событий
        webview.loadFrameScript(Qt.resolvedUrl("../shared-js/framescript.js"));
        webview.addMessageListener("webview:action")
    }

    onRecvAsyncMessage: {
        switch (message) {
        case "webview:action":
            // Получение асинхронных ответов
            break
        }
    }

    Component.onCompleted: {
        // Отключение CORS
        WebEngineSettings.setPreference("security.disable_cors_checks", true, WebEngineSettings.BoolPref)
    }
}
</code></pre><p>В WebView доступен метод <a href="https://developer.auroraos.ru/doc/software_development/reference/webview/qml-sailfishos-webview-webview#runJavaScript-method" rel="noopener noreferrer nofollow"><code>runJavaScript</code></a>. Метод запускает фрагмент JavaScript в контексте загруженного документа <a href="https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model/Introduction" rel="noopener noreferrer nofollow">DOM</a>. С его помощью можно добавить функцию в QML-компонент, которая выполнит запрос к JavaScript-функции и подготовит данные для получения ответа со стороны JavaScript-библиотеки.</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/pages/KMMAgent.qml</code></p><pre><code>property var stateResponse: ({})

function run(method, result, error) {
    // По ключевому слову функция определит тип запроса: асинхронные данные или нет
    if (method.indexOf("return") === -1) {
        // Далее будет добавлен функционал, который возвращает ключ асинхронного запроса
        webview.runJavaScript("return " + method, function(key) {
            // Следует запомнить функции, чтобы выполнять их после ответа
            stateResponse[key] = [result, error]
        }, error);
    } else {
        // Запуск по умолчанию, синхронные функции не требуют вмешательства
        webview.runJavaScript(method, result, error);
    }
}
</code></pre><p>Получить событие можно в методе WebView <code>onRecvAsyncMessage</code>. При выполнении асинхронной функции QML-компонент (KMMAgent) выполняет запрос к синхронной функции Wrapper, которая производит запуск асинхронной функции KMP и возвращает уникальный ключ. Сохраняется пара QML-функция и уникальный ключ. При получении события вызывается соответствующая QML-функция по ключу. Данные события можно получить через переменную <code>data</code>. Обработка в <code>onRecvAsyncMessage</code> представлена ниже:</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/pages/KMMAgent.qml</code></p><pre><code>onRecvAsyncMessage: {
    switch (message) {
    case "webview:action":
        try {
            // Первое событие, которое обозначает готовность компонента
            if (data.caller === 'init') {
                root.completed()
            // Другие события
            } else if (root.stateResponse[data.caller] !== undefined) {
                if (data.response.hasOwnProperty('stack')) {
                    // Обработка ошибки
                    root.stateResponse[data.caller][1](data.response.message)
                } else {
                    // Обработка ответа
                    root.stateResponse[data.caller][0](data.response)
                }
            }
        } catch (e) {
            // Общая ошибка запроса, парсинга данных
            root.stateResponse[data.caller][1](e.toString())
        }
        break
    }
}
</code></pre><p>QML готов к выполнению функций Kotlin Multiplatform. Через событие <code>init</code> будет получен сигнал от WebView о готовности компонента KMMAgent к работе. Теперь в приложении ОС Аврора есть возможность выполнять запросы к Wrapper. На главной странице приложения в компоненте QML ApplicationWindow нужно инициализировать KMMAgent, тогда его можно будет вызвать на всех дочерних страницах:</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/RSSReader.qml</code></p><pre><code>// Инициализация на странице приложения
KMMAgent {
	id: agent
}

// Запуск асинхронной функции
agent.run(
    "shared.Service.get.getAllFeeds()",
    function(response) {
        console.log(response)
    },
    function(error) {
        console.log(error)
    }
)

// Запуск синхронной функции
agent.run(
    "return shared.Service.get.getAllFeeds()", 
    function(response) { 
        console.log(response)
    },
    function(error) {
        console.log(error)
    }
)

// Получение данных, которые могут содержать переменные KMP
agent.run(
    "return shared.AppConstants.links.API_URL", 
    function(response) { 
        console.log(response)
    },
    function(error) {
        console.log(error)
    }
)
</code></pre><h3>Wrapper</h3><p>Взаимодействие Kotlin Multiplatform и QML обеспечивает библиотека Wrapper. В ней будет полный доступ к модулю Kotlin Multiplatform, что позволит подготовить данные для QML. Библиотеку нужно подключить в точку входа — файл <code>index.html</code>, который будет загружен в QML-компонент <code>KMMAgent</code>. В <code>index.html</code>, расположенный в директории <code>qml</code> приложения ОС Аврора, нужно подключить библиотеку Wrapper и отправить событие с информацией о том, что инициализация WebView успешно завершилась. Это позволит точно определить готовность WebView. Содержимое файла <code>index.html</code> будет следующим:</p><p>Файл <code>&lt;проект&gt;/auroraApp/RSSReader/qml/shared-js/index.html</code></p><pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;

&lt;head&gt;
	&lt;meta charset="utf-8"&gt;
	&lt;title&gt;KMP&lt;/title&gt;
	&lt;script src="./dist/shared.js"&gt;&lt;/script&gt;
&lt;/head&gt;

&lt;body&gt;
    &lt;p id="demo"&gt;&lt;/p&gt;
    
	&lt;script type="text/javascript"&gt;
		function init() {
			async function init() {
				// Проверка работоспособности библиотеки
				document.getElementById("demo").innerHTML = shared.Helper.randomUUID()
				// Отправка события готовности
				shared.Helper.sendEvent("init")
			}
			// Здесь можно добавить задержку отправки готовности 
			setTimeout(init, 0)
		}
		init()
	&lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
</code></pre><p>Создать в корне проекта директорию для библиотеки Wrapper и добавить в неё <code>webpack.config.js</code> для сборки.</p><p>Файл <code>&lt;проект&gt;/auroraApp/wrapper/webpack.config.js</code></p><pre><code>const path = require('path');

В <code>index.js</code> добавить экспорт нужных компонентов, к которым можно обратиться из QML.</p><p>Файл <code>&lt;проект&gt;/auroraApp/wrapper/src/index.js</code></p><pre><code>export * from './Helper';
export * from './Service';
</code></pre><p>Дальше c Wrapper можно работать как с обычной библиотекой JS, и вызывать все нужные функции в QML.</p><p>Объект Helper (вспомогательный класс) поможет создать уникальный ключ и отправит событие после ответа асинхронной функции Kotlin Multiplatform.</p><p>Файл <code>&lt;проект&gt;/auroraApp/wrapper/src/Helper.js</code></p><pre><code>import {v4 as uuidv4} from 'uuid';

export const Helper = {
    // Создать случайный UUID
    randomUUID: function () {
        return uuidv4()
    },
    // Обертка с событием отправки после получения данных
    request: function (fun, callback, delay) {
        const caller = Helper.randomUUID()
        setTimeout(async () =&gt; {
            try {
                Helper.sendEvent(caller, callback(await fun()))
            } catch (e) {
                Helper.sendEvent(caller, e)
            }
        }, delay)
        return caller
    },
    // Отправить событие
    sendEvent: function (caller, response) {
        const customEvent = new CustomEvent("framescript:log", {
            detail: {
                response: response,
                caller: caller
            }
        });
        document.dispatchEvent(customEvent);
    }
}
</code></pre><p>Helper упростит написание обёртки на асинхронные функции Kotlin Multiplatform. Код обёртки будет выглядеть следующим образом:</p><p>Файл <code>&lt;проект&gt;/auroraApp/wrapper/src/Service.js</code></p><pre><code>import shared from "shared";
import {Helper} from "./Helper";

const JsRssReader = new shared.com.github.jetbrains.rssreader.core.JsRssReader()

export const Service = {
    get: {
        getAllFeeds: function (forceUpdate = true) {
            return Helper.request(async () =&gt; {
                return await JsRssReader.getAllFeedsPromise(forceUpdate)
            }, (response) =&gt; {
                return response.toArray();
            }, 0 /** задержка, если это необходимо **/)
        },
    }
}
</code></pre><p>Выполнить сборку библиотеки Wrapper:</p><pre><code class="bash">npm run build
</code></pre><p>Собранную библиотеку Wrapper нужно добавить в папку <code>qml</code> и подключить её в <code>index.html</code>.</p><pre><code>&lt;head&gt;

</code></pre><p>В итоге получится три независимых проекта, объединенных в приложение на платформе ОС Аврора.
